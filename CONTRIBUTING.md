## 🌟 Ways to Contribute

### 1. Add New Resources
Found a great course, book, tool, or paper? Add it!
- Make sure it fits an existing section (or propose a new one)
- Follow the formatting guidelines below

### 2. Fix Broken Links
Links rot over time. If you find a broken one:
- Open an issue using the **Broken Link** template, or
- Submit a PR with the fix

### 3. Improve Descriptions
- Clarify existing descriptions
- Add context about why a resource is valuable
- Update outdated information

### 4. Suggest New Sections
If there's a topic area we're missing, open an issue with the **Suggest Section** template.

### 5. Share Your Experience
- Add interview tips from your own experience
- Contribute sample questions you've encountered
- Share salary data points

## ✅ Resource Quality Standards

Before adding a resource, make sure it meets these criteria:

| Criteria | Requirement |
|----------|------------|
| **Relevance** | Directly useful for AI/ML job seekers |
| **Quality** | Well-regarded in the community |
| **Accessibility** | Free or has clear pricing info |
| **Maintenance** | Currently maintained or timeless (classic textbooks, seminal papers) |
| **No spam** | No self-promotion without genuine value |
| **Working links** | All URLs must be valid and accessible |

### For SaaS/Platform Resources
When adding any SaaS product or platform, you **must** include:
- Whether a free tier exists
- Free tier limitations
- Paid plan pricing
- Present this in a table format

## 📐 Formatting Standards

### Links
```markdown
[Resource Name](https://url.com/) — Brief description
```

### Tables (for platforms/tools)
```markdown
| Platform | Free Tier | Paid Plans | Best For |
|----------|-----------|------------|----------|
| [Name](https://url.com/) | ✅ Free / ❌ No free tier | $X/mo | Use case |
```

### Papers
```markdown
| Year | [Paper Title](https://arxiv.org/abs/XXXX.XXXXX) | Key contribution |
```

### Books
```markdown
| [Book Title](https://url.com/) | Author(s) | Level | ✅ Free / ❌ Paid |
```

## 🔄 Pull Request Process

1. **Fork** this repository
2. **Create a branch** (`git checkout -b add-resource-xyz`)
3. **Make your changes** following the formatting guidelines
4. **Verify all links** are working
5. **Submit a PR** using the PR template

### PR Checklist
- [ ] All links are verified and working
- [ ] Proper formatting is followed
- [ ] Pricing info included for SaaS/platforms
- [ ] Resource fits the appropriate section
- [ ] Description is clear and concise
- [ ] No duplicate entries

## 🐛 Reporting Issues

Use the appropriate issue template:
- **🔗 Broken Link** — Report a dead or moved link
- **📦 Add Resource** — Suggest a new resource to add
- **💡 Suggest Section** — Propose a new topic section
